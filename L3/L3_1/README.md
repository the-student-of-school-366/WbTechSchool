# DelayedNotifier — отложенные уведомления через очереди

Во многих сервисах пользователям нужно напоминать о чём-то не сразу, а через какое-то время. Это могут быть уведомления о предстоящих встречах, напоминания о заказе, сообщения в мессенджере или email через несколько часов или дней. DelayedNotifier — это бэкенд-компонент, который автоматизирует этот процесс: он принимает запросы, «запоминает» время отправки и доставляет сообщения точно в срок.

Сервис принимает уведомления, сохраняет их в очередь через RabbitMQ и обрабатывает в фоновом режиме. В случае сбоев предусмотрен механизм повторных попыток.

## Требования

### Встроенные HTTP-методы

*   **POST /notify** — создание уведомления с указанием даты и времени отправки.
*   **GET /notify/{id}** — получение текущего статуса конкретного уведомления.
*   **DELETE /notify/{id}** — отмена ранее запланированного уведомления.

### Фоновая обработка очереди

*   Отправка уведомлений строго в указанное пользователем время.
*   Автоматический повтор отправки при возникновении ошибок (с использованием экспоненциальной задержки).

## Дополнительные возможности

*   **Кэширование:** Использование Redis для хранения состояний и быстрой проверки статуса уведомлений.
*   **Мультиканальность:** Поддержка различных каналов доставки сообщений (например, Email или Telegram).
*   **Пользовательский интерфейс:** Наличие простого UI (HTML + JS) с формой создания задач и таблицей текущих статусов для визуального тестирования системы.
